## 📋 **测试流程（30分钟/人）**

### **阶段1: 前测 (5分钟)**
1. 简短介绍游戏目的
2. 询问用户对韩语的了解
3. 记录初始预期

### **阶段2: 自由探索 (10分钟)**
- 打开游戏链接
- 让用户自己操作（不提示）
- 观察：
  - 是否能找到"开始学习"
  - 是否理解拖拽操作
  - 是否使用TTS按钮
  - 遇到什么困惑

### **阶段3: 任务完成 (10分钟)**
- 要求用户完成Level 1（5个字母学习+拼图）
- 观察：
  - 用时多少
  - 错误次数
  - 是否需要帮助
  - 情绪反应（沮丧/兴奋）

### **阶段4: 后测访谈 (5分钟)**
1. **评分**: "你给这个游戏打几分（1-10）？"
2. **最有用**: "哪个功能对你学习最有帮助？"
3. **最困难**: "哪个部分你觉得最难？"
4. **改进建议**: "你想增加什么功能？"
5. **推荐意愿**: "你会推荐给朋友吗？（0-10分）"

## 📊 **数据收集指标**

| 指标 | 测量方法 | 目标值 |
|------|----------|--------|
| 任务完成率 | 完成Level 1的用户比例 | > 90% |
| 平均用时 | 从开始到完成的时间 | < 10分钟 |
| 平均错误次数 | 拖拽错误的次数 | < 5次 |
| TTS使用频率 | 点击发音按钮次数 | > 3次/用户 |
| 推荐指数 (NPS) | "推荐给朋友"评分 | > 7分 |
| 学习信心提升 | "学完后感觉能记住"评分 | > 6分 |

---

## 🐛 **常见问题预判**

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 用户不知道可以拖拽 | UI提示不足 | 增加新手引导动画 |
| TTS发音不清晰 | 浏览器TTS质量差 | 集成sag专业发音 |
| 字母颜色混淆 | 配色区分度不够 | 增加图案辅助 |
| 手机操作困难 | 按钮太小 | 增大触摸区域（mobile.css已优化） |
| 不知道已完成 | 完成提示不明显 | 增强CompleteScene动画 |

## 🔄 **迭代计划**

1. **如果任务完成率 < 80%** → 简化操作流程
